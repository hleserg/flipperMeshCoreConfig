# Передача дел 🐢

Привет. Это короткая записка для тебя-уставшего или для того, кто откроет репозиторий без всякого контекста. Читается сверху вниз, ничего держать в голове заранее не нужно.

**Что это вообще.** Приложение для Flipper Zero, которое настраивает и использует
LoRa-меш-ноды **MeshCore**. Радио — на нодах, не на флиппере. Флиппер говорит с
нодой по проводу (UART, пины 13/14/GND) на языке companion-протокола MeshCore.
Три режима: настройка радио, мессенджер, полевой логгер.

Условные обозначения ниже:

- **[✓ СДЕЛАНО]** — уже работает, проверено. Трогать не надо.
- **[→ ТВОЙ ХОД]** — ждёт живого человека (пальцы, паяльник, железо из Китая).

## Когда приедут ноды

1. **[→ ТВОЙ ХОД] Прошить.** Роверы Сергея — нашей UART-сборкой (веб-флешер),
   репитеры и ноды Марка — официальным флешером. Подробно — в гайде Сергея на
   сайте документации.
2. **Настроить радио.** На флиппере: **Connect** к ноде → **Profiles** → выбрать
   пресет → **Apply**. Галочки покажут, что нода реально приняла настройки
   (`[ok]` — приняла и подтвердила, `[~]` — приняла, но подтвердить не может,
   `[!!]` — сказала ok, но не изменилась).
3. **Полевой тест.** По гайдам на сайте — Сергей с флиппером, Марк с телефоном,
   совместная часть (сплав по реке) одна на двоих.

## Если что-то сломалось

- **Флиппер перестал отвечать по CLI** — скорее всего завис сервис хранилища
  (нельзя читать CSV, пока логгер держит файл открытым). Лечится Back+Left.
- **Девборд не виден в системе** — он не отдаёт свой USB в комп через флиппер;
  для заливки нужен BOOT+RESET (ROM-режим), см. `tools/node-emulator/devboard-bridge/`.
- **FAP не ставится** — флиппер должен быть на Unleashed (наш SDK под него,
  API 87.8); проверь `ufbt` индекс.

Полный разбор граблей — в гайдах на сайте (раздел «🪤 ГРАБЛИ») и в `AGENTS.md`.
