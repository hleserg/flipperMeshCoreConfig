MeshCore: настройка и полевой тест — инструкция для Марка 🐢

Мы объединяем железо: в сети будет 8 нод. Твои три настраиваешь ты (эта инструкция), остальные пять — Сергей. Документ самодостаточный, ничего другого читать не надо.

Твои ноды и их роли

Твоя нодаИмя в сетиРольПрошивка
V4 (ESP32-S3)ROVER-M🎒 Автологгер — ходит с тобой, сам пишет данныеcompanion_radio_wifi

Зачем тебе два ровера в кармане. У них разные задачи: V4 — это измерительный прибор (к нему цепляется телефон по WiFi, крутится Python-логгер, пишутся CSV). T114 — это то, что мы, собственно, тестируем: маленькая, экономичная, именно такая будет у людей на фесте. Заодно на ней меряем реальный расход батареи за день.

ЧАСТЬ 1. ДОМА — НАСТРОЙКА ТВОИХ ТРЁХ НОД 🔧

Делается один раз, за столом, с компьютера по USB.

⚠️ Разнеси ноды на 2–3 метра, пока настраиваешь. Передатчики по 20+ dBm в 20 см друг от друга перегружают приёмники, и ты словишь «глюки связи», которых в реальности нет.

1.0 Поставить CLI на компьютер

pipx ставить не надо, хватает обычного pip.

Windows (PowerShell):

py --version                              # нет Python? → winget install Python.Python.3.12
py -m pip install --user meshcore-cli
meshcli -v

macOS / Linux:

python3 -m pip install --user meshcore-cli
meshcli -v

Если meshcli не находится — папка Scripts/bin не в PATH. Не воюй с PATH, запускай через модуль, работает идентично:

py -m meshcore_cli <аргументы>        # Windows
python3 -m meshcore_cli <аргументы>   # mac/Linux

Найти порт ноды — в диспетчер устройств лезть не нужно:

meshcli -l

Порт на Windows выглядит как COM3/COM5, на mac/Linux — /dev/tty.usbserial-* или /dev/ttyACM0. Дальше в инструкции пишу COM5 — подставляй свой.

  • V4 не появился в списке → доставь драйвер CP210x (у ESP32-плат обычно он).
  • T114 (nRF52840) определяется сам, драйвер не нужен.

1.1 Прошить все три

Веб-флешер: hleserg.github.io/meshcore-node-fw — прямо из браузера, компилятор не нужен.

НодаПрошивка
V4companion_radio_wifi ← именно WiFi-вариант, без него логгер не заработает
T114 №1companion (BLE-вариант)
T114 №2repeater

1.2 🚨 Радио — самое важное. Цифры менять нельзя

Если у твоих нод и у нод Сергея радио-настройки отличаются хоть на йоту — ноды физически не услышат друг друга, и мы полдня будем искать несуществующую поломку. Ставь ровно это, на все три:

freq = 869.525 МГц    BW = 250 кГц    SF = 11    CR = 5
meshcli -s COM5 set radio 869.525,250,11,5

💡 Синтаксис set слегка разнится между версиями прошивки. Не гадай — набери set help в CLI ноды и посмотри список. Либо выстави радио в мобильном приложении: там просто поля, ошибиться негде.

Проверить, что применилось:

meshcli -s COM5 infos

Сверь radio_freq, radio_bw, radio_sf, radio_cr с цифрами выше.

1.3 Дать имена

Ровно эти, они попадут в CSV и по ним мы будем сводить данные:

meshcli -s COM5 set name ROVER-M      # твой V4
meshcli -s COM5 set name POCKET-M     # твой T114 №1
meshcli -s COM5 set name REP3         # твой T114 №2

Наклей стикеры с именами на корпуса — в поле, когда нод восемь, это спасает.

1.4 Пароль на REP3

У репитера есть админ-CLI, доступный по радио, и дефолтный пароль общеизвестен. Поставь свой, иначе любой желающий на фесте перенастроит сеть. Репитер настраивается своим CLI: meshcli -s COM5 -r, дальше команды. Пароль запиши и передай Сергею — репитер уходит в общую инфраструктуру, управлять им может понадобиться обоим.

1.5 Синхронизировать часы

meshcli -s COM5 clock sync

На всех трёх. Это критично: мы будем сводить логи с двух телефонов, и если время разъедется — данные не склеятся.

1.6 Познакомить ноды и передать контакты Сергею

Включи все три рядом (помним про 2–3 метра), дай каждой отправить advert:

meshcli -s COM5 advert
meshcli -s COM5 contacts     # каждая должна видеть две других

Чтобы в поле не ждать обнаружения, перешли Сергею свои контакты заранее:

meshcli -s COM5 export_contact

Выдаст URI вида meshcore://... — кинь все три в телеграм. Он их импортирует (import_contact <URI> или сканом QR в приложении), а тебе пришлёт свои пять.

1.7 Зарядить

Всё под завязку + повербанк. V4 в режиме WiFi-точки жрёт заметно больше остальных — ему повербанк обязателен, а не «на всякий случай».

✅ Чек-лист части 1: прошито 3/3 · радио 869.525/250/11/5 на всех трёх · имена ROVER-M/POCKET-M/REP3 · пароль на REP3 задан и передан · часы синхронизированы · контакты выгружены и отправлены · всё заряжено.

ЧАСТЬ 2. ДОМА — НАСТРОЙКА ТЕЛЕФОНА 📱

Тебе нужны два приложения на одном телефоне: Termux (логгер для V4) и клиент MeshCore (для T114).

2A. Termux — логгер для ROVER-M

2.1 Ставить Termux с F-Droid, не с Google Play

Версия в Play исторически отставала и ломалась. Плюс отдельно поставь приложение Termux:API (оттуда же).

2.2 Зеркало и ПОЛНОЕ обновление — в этом порядке

Сначала выбери быстрое зеркало, иначе будешь качать 86 МБ на скорости 150 кБ/с:

termux-change-repo

Стрелками: Mirror group → свой регион → OK.

Теперь обновляем всё и только потом ставим:

pkg upgrade                 # ← именно upgrade, не update!
pkg install python termux-api clang

🚨 Кочка, на которую мы уже наступили. pkg update ≠ pkg upgrade. update только скачивает списки пакетов, ничего не обновляя. Если после него поставить python, получишь новый Python поверх старых библиотек — а Termux это rolling-release, там так нельзя. Выглядит так:

ImportError: dlopen failed: cannot locate symbol "XML_SetHashSalt16Bytes"

Лечится тем же pkg upgrade — дообновить всё, соглашаясь на все вопросы.

2.3 Если pip ругается bad interpreter

bash: .../bin/pip: .../bin/python3.13: bad interpreter: No such file or directory

Termux поднял Python на мажорную версию, а скрипт pip остался от старой.

python -m ensurepip --upgrade
python -m pip --version

Дальше везде вызывай python -m pip, а не голый pip. Если ensurepip падает с тем же dlopen — сначала доделай pkg upgrade.

2.4 Ставим библиотеку — сразу через --no-deps

Обычный pip install meshcore на Termux упадёт, и это закономерно: он тянет bleak (Bluetooth), а тот на Linux/Android требует dbus-fast, который надо компилировать с dbus-заголовками. На Termux это не собирается.

bleak нам не нужен — логгер работает по TCP. Проверено, без него всё живёт:

python -m pip install meshcore --no-deps
python -m pip install pycayennelpp pycryptodome pyserial-asyncio-fast

pycryptodome будет пару минут собираться из исходников — это норма, для того и ставили clang. Он обязателен: без него meshcore не импортируется вообще.

В конце pip напишет красным: ERROR: ... meshcore 2.3.7 requires bleak, which is not installed Это не ошибка. Ниже будет Successfully installed .... pip просто ворчит, что мы намеренно пропустили bleak. Игнорируй.

Проверка:

python -c "from meshcore import MeshCore; print('ok', hasattr(MeshCore,'create_tcp'))"

Должно быть ok True.

2.5 Закинуть meshlog.py в Termux

Скрипт должен лежать в домашней папке Termux (~), а не в общей памяти.

Дать доступ к памяти (один раз, Android спросит разрешение):

termux-setup-storage

Найти скачанный файл — не гадай, где он, просто найди:

find ~/storage/shared -name "meshlog.py" 2>/dev/null
Откуда качалГде искать
Браузер / «Сохранить в Загрузки»~/storage/downloads/
Telegram (Android 11+)~/storage/shared/Android/media/org.telegram.messenger/Telegram/Telegram Documents/
Telegram (старые версии)~/storage/shared/Telegram/Telegram Documents/

Скопировать к себе (кавычки обязательны — в пути Telegram есть пробелы):

cp "$(find ~/storage/shared -name 'meshlog.py' 2>/dev/null | head -1)" ~/
ls -l ~/meshlog.py

Запасной вариант — открой файл как текст, скопируй целиком, потом:

termux-clipboard-get > ~/meshlog.py

Проверить:

cd ~
python meshlog.py --help

📁 Запускай логгер всегда из домашней папки (cd ~). Скрипт пишет CSV в ./meshtest относительно текущего каталога. Забрать логи потом: cp -r ~/meshtest ~/storage/downloads/

2.6 Разрешить геолокацию

termux-location

Первый раз Android спросит разрешение — дай. Должен вернуться JSON с координатами. Если пропустить — в CSV не будет позиций, и карту покрытия не построить.

2.7 Победить засыпание и мобильный интернет

  • termux-wake-lock перед запуском — иначе Android усыпит Termux в кармане.
  • В настройках батареи сними с Termux все ограничения энергосбережения.
  • Отключи мобильные данные на время прогона. Точка доступа V4 без интернета,

и Android может увести трафик в LTE — тогда логгер не достучится до ноды.

2.8 Генеральная репетиция дома (не пропускай!)

Включи V4, подключись телефоном к его WiFi, посмотри IP на экранчике V4:

cd ~
python meshlog.py --tcp 192.168.4.1:5000 --ping REP3

Убедись, что подключается, в консоли бегут ping REP3 #N: OK ... ms, и в ~/meshtest появились CSV и растут.

Порт не отвечает — попробуй 4000. IP бери с экрана ноды, не угадывай.

ЧАСТЬ 3. В ПОЛЕ — сплав по реке 🛶

Что взять

Три ноды (подписанные) · телефон · повербанки · кабели · пароль от REP3 · гермомешок и зип-пакеты · оффлайн-карты, скачанные заранее.

Как ты пишешь лог

Всё как раньше, инструмент не изменился:

termux-wake-lock
cd ~
python meshlog.py --tcp 192.168.4.1:5000 --ping BASE,REP1,REP2 --interval 15

Метки в логе без рук: отправь с POCKET-M сообщение в канал вроде TOCHKA povorot. Оно прилетит на оба ровера и запишется в events.csv с временем — потом сопоставишь с координатами.

Точечные трейсы. В ключевых точках с приложения: Tools → Trace Path до каждой фиксированной ноды. SNR — главное число, сразу видно слабое звено.

Полевая часть переехала с ровной знакомой площадки на сплав по реке через незнакомый лес. Это меняет не инструменты, а сам протокол теста.

Четыре следствия, которые ломают старый план

1. Местность неизвестна. Точки по карте заранее не расставить. Методика становится адаптивной: пришёл → осмотрелся → поставил → замерил → решил.

2. Топология линейная. Река — это линия, а не полигон. Вопрос не «покрыта ли площадка», а на сколько километров русла добивает связь и что делают повороты.

3. Движение одностороннее. Вниз по течению не вернуться. Репитер, оставленный выше по реке, забыт навсегда. Вокруг этого ограничения строится вся программа.

4. Лес глушит. Листва на 868 МГц поглощает заметно сильнее открытого поля, мокрая — ещё сильнее. Цифры будут хуже фестивальных. Это условия, а не поломка.

🚨 Безопасность — прочитать до выезда

🚨 Сеть под тестом — не средство связи. Её испытывают, она может замолчать в любой момент, и именно это вы и проверяете. Не стройте на ней безопасность: о свистках, визуальных сигналах, времени и месте встречи договаривайтесь до расхождения, голосом.

  • Перед любым расхождением — точка сбора и жёсткий контрольный срок возврата.
  • В незнакомый лес поодиночке далеко не уходить. Оффлайн-карты скачать заранее,

компас или навигатор с собой.

  • Каждую поставленную ноду фиксировать: координаты, фото места, описание

ориентира. В незнакомом лесу ноду без метки не находят — это проверено.

  • Вода убивает электронику. Нода в зип-пакете внутри гермы; кабели и логгер

туда же. Всё, что не в руках, привязано к судну.

  • Энергия. Розеток нет несколько дней. Посчитай бюджет повербанков заранее;

солнечная панель для базы на стоянке — большой плюс.

  • GPS под плотным пологом и в узкой долине врёт или пропадает. Пустые

lat/lon в логе посреди леса — норма, а не повод паниковать.

Кто что делает

Тест парный, и в этом его главная ценность: два логгера пишут одновременно, в разных точках, по одним и тем же часам.

СергейМарк
Инструмент операторафлиппер с картой памятителефон
Свои нодыROVER-S, POCKET-SROVER-M, POCKET-M, REP3
Ставит стационарныеBASE, REP1, REP2держит REP3 в резерве
Пишет логина SD флипперав Termux на телефоне

Инструменты разные, сценарии ниже — одни и те же. Их выполняют вдвоём, и именно поэтому оба лога потом сводятся в одну картинку.

⚠️ Сверьте часы перед первым замером и запишите время старта. Два лога с разъехавшимся временем свести невозможно — это единственная ошибка, которую нельзя починить постфактум.

Две программы вместо обхода периметра

Полевая работа делится на две принципиально разные части.

🏕️ Программа «Лагерь» — основной объём тестов

Делается на каждой ночёвке. Ноды разворачиваются и собираются обратно, поэтому здесь можно не бояться потерь и мерить сколько угодно.

Развёртывание, вечером:

  1. BASE — на самую высокую доступную точку рядом с лагерем: высокий берег,

бугор, дерево.

  1. Репитеры — по кандидатным точкам вокруг: за перегибом рельефа, на повороте

реки, в стороне от лагеря.

  1. Перекличка. Оба включают роверов, дают advert и проверяют, что видят все

ноды сети. Не видно чью-то — разбираемся сейчас, а не через два часа.

Что мерить:

  • расхождение в лес от лагеря — до потери связи;
  • вверх и вниз по берегу — линейная дальность вдоль русла;
  • за перегиб рельефа — работает ли репитер там, где нет прямой видимости;
  • ровер ↔ ровер между двумя людьми.

Ровер ↔ ровер — главный сценарий феста

На фесте люди будут переписываться друг с другом, а не с базой. Проверяем именно это, и это самый важный тест всего выезда:

  1. Расходитесь на максимальное расстояние — по берегу, в лес, в разные стороны.
  2. Пишете друг другу и ждёте доставки.
  3. Специально ищете позиции, где прямой связи заведомо нет: за перегибом, за

поворотом, в низине. Проверяем, что меш реально роутит через репитеры, а не «работает, пока видим друг друга».

Опорная точка + ходок

Один встаёт на известную точку и стоит, второй ходит. Зачем: если сигнал просел, надо понимать — это позиция ухудшилась или канал деградировал целиком (дождь, помеха, севшая батарея). Неподвижный ровер даёт эталон для сравнения. Потом поменяйтесь ролями.

Закрыть дыру — вот зачем нужен REP3

Из замеров находим мёртвую зону. Ставим туда (или на точку, которая её перекрывает) REP3, включаем, ждём advert — и проходим этот кусок заново.

Смысл: это не «поставили и надеемся», а нормальный эксперимент — замер → вмешательство → повторный замер. Закрылась дыра — цифры покажут. Не закрылась — двигаем REP3 и повторяем.

✅ Утром собрать всё до единой ноды. Внеси это отдельным пунктом в чек-лист сборки лагеря — забытая на дереве нода не догонит вас по течению.

Повторяемость — главная ценность этой программы: одинаковые тесты на разных стоянках дают сравнимые между собой цифры.

🛶 Программа «Сплав» — движение

🚨 На воде на берегу не оставляем ничего. Вернуться нельзя. Всё железо плывёт с вами.

Логгеры пишут непрерывно, и трек сам рисует карту покрытия — река делает обход за тебя, специально ходить никуда не нужно.

Главный сценарий: два судна расходятся по воде и переписываются на растущей дистанции. Это та же проверка «ровер ↔ ровер», только дистанция набирается сама.

Отмечай характерные места: поворот, сужение, высокий берег, порог, выход на плёс.

💡 Бонус, которого на фесте не будет: прямой плёс — идеальная трасса прямой видимости над водой. Сделайте отдельный мини-эксперимент: расходитесь по прямой воде, пока связь не порвётся, зафиксируйте дистанцию и SNR. Это верхняя граница возможностей вашего железа — эталон, с которым потом сравнивать всё остальное.

Единственный случай, когда репитер ставится «в одну сторону»

Если очень хочется проверить, пробивает ли репитер за поворот реки — ставь его только там, где гарантированно пройдёшь мимо ещё раз или дойдёшь пешком по берегу.

🚨 Во всех остальных случаях — не ставь. Нода, оставленная ниже по течению от вашей будущей стоянки, потеряна. Это не «потом заберём», это «навсегда».

Как ставить ноды в незнакомой местности

Правила вместо точек на карте — работают там, где ты никогда не был.

  • Высота важнее расстояния. Метр подъёма часто даёт больше, чем сто метров

сближения. Верх берега, бугор, ветка дерева. Никогда на землю и не на дно лодки.

  • Антенна вертикально с обеих сторон. Косо лежащая антенна теряет очень много.
  • Держи ноду в стороне от тела. Человек хорошо поглощает 868 МГц — на фесте

тысячи тел будут главным фактором, привыкай сразу.

  • Поворот реки = стена. Прямой видимости нет, сигнал идёт отражением и

дифракцией. Репитер осмысленно ставить на самом повороте, на высоком внешнем берегу — оттуда он видит оба плеча сразу.

  • Плотный ельник и мокрая листва глушат сильнее сухого лиственного леса.

Записывай, какой был лес, иначе цифры потом не с чем сравнивать.

  • Сначала замерь, потом оставляй. Поставил → дал advert → проверил SNR →

и только потом решай, оставлять ли ноду здесь.

Как искать дальность

Процедура вместо «обойди периметр» — годится для любой незнакомой местности.

  1. Отходишь или отплываешь шагами, останавливаясь каждые **N метров на 30–60

секунд**, чтобы набралось несколько замеров.

  1. Смотришь на живой SNR. Как только приблизился к порогу — ставишь метку.
  2. Отошёл ещё немного, связь порвалась — вернулся к последней хорошей точке.

Это и есть граница для этих условий.

  1. Записал: дистанция, тип леса, рельеф, была ли прямая видимость.

Порядок величины, которого стоит ожидать: густой лес — сотни метров; открытая вода по прямой — километры.

⚠️ Это ожидание для проверки, а не обещание. Твой лес может оказаться вдвое хуже. Смысл упражнения — получить свои цифры, а не подтвердить чужие.

---

⚠️ Дистанция на реке двусмысленна — записывай обе

Радио интересует прямая линия, а не русло. На извилистой реке 3 км сплава могут оказаться 400 м по прямой.

⚠️ В журнале фиксируй и километры по воде, и прямое расстояние (по координатам). Иначе выводы будут просто неверными.

Три поправки, без которых цифры соврут:

  • Лес жёстче фестивального поля. Прошло в лесу — на открытой площадке будет

лучше.

  • Но фест добавляет то, чего в лесу нет: тысячи человеческих тел, генераторы,

толпа с телефонами. Поэтому запас +5…+10 dB остаётся обязательным.

  • Дальность над водой не переносится на фест напрямую. Прямой плёс — это

лучший случай, а не типичный. Ставить по нему планы нельзя.

Мини-словарик

  • SNR (dB) — насколько сигнал торчит над шумом. Главная метрика качества.

Для LoRa минус — норма, она вытягивает сигнал из-под шума примерно до −20 dB.

  • RSSI (dBm) — просто громкость сигнала. Ближе к нулю = сильнее.
  • RTT — время «туда-обратно» до подтверждения. ACK — само подтверждение.
  • advert — маячок, которым нода объявляет о себе в сети.
  • Trace Path — MeshCore-аналог traceroute: гоняет тест-пакет по маршруту и

возвращает SNR и время по каждому прыжку.

  • Companion / Repeater — роли: клиент (к нему цепляется телефон) / ретранслятор.
  • SF / BW / CR — крутилки модема LoRa (дальность ↔ скорость). У всех нод сети

должны совпадать.

  • Хоп — прыжок через промежуточную ноду. Мультихоп = цепочка ретрансляторов.
  • ISM — диапазоны для связи без лицензии (868 и 433 МГц).
